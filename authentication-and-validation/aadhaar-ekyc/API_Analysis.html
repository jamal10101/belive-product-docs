API Analysis & Parameters

UIDAI Aadhaar eKYC API Documentation · Endpoints, Parameters, Responses, Error Codes

API Provider Information

Provider: UIDAI (Unique Identification Authority of India)

Access via: Licensed ASA (Authentication Service Agency) or KUA (KYC User Agency) — e.g., Digio, Signzy, Protean

Sandbox URL: https://stage1.uidai.gov.in

Production URL: Provided after licensing approval

Authentication: Digital signature certificate + licensed sub-AUA code

Rate Limit: Varies by license tier (typically 100-1000 requests/day for sandbox)

API Endpoint 1: Generate OTP

POST /api/v1/otp/generate

Sends OTP to the mobile number linked with the given Aadhaar number.

Request Parameters

ParameterTypeRequiredDescriptionValidation
uidStringYes12-digit Aadhaar numberExactly 12 digits, numeric only, Verhoeff checksum
txnIdStringYesUnique transaction IDUUID v4 format
appIdStringYesLicensed application identifierAssigned by UIDAI
timestampDateTimeYesRequest timestampISO 8601 (e.g., 2026-03-22T10:30:00Z)
consentStringYesUser consent flagMust be "Y" — user must explicitly consent

Response — Success

{
  "status": "SUCCESS",
  "txnId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "message": "OTP sent to registered mobile number",
  "otpSentTo": "XXXXXXXX90"   // last 2 digits shown
}

Response — Error

{
  "status": "FAILURE",
  "errorCode": "K-200",
  "message": "Invalid Aadhaar number",
  "txnId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
}

API Endpoint 2: Verify OTP & Get eKYC Data

POST /api/v1/otp/verify

Verifies the OTP entered by user and returns demographic eKYC data.

Request Parameters

ParameterTypeRequiredDescriptionValidation
uidStringYes12-digit Aadhaar numberSame as OTP generate request
txnIdStringYesTransaction ID from OTP requestMust match generate response txnId
otpStringYes6-digit OTP entered by userExactly 6 digits, numeric
consentStringYesUser consent flagMust be "Y"
timestampDateTimeYesRequest timestampISO 8601

Response — Success (eKYC Data)

{
  "status": "SUCCESS",
  "txnId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "ekyc": {
    "name": "Atharva Sunil Patil",
    "dob": "2001-05-15",          // YYYY-MM-DD format
    "gender": "M",                 // M = Male, F = Female, T = Transgender
    "photo": "base64_encoded...",  // Aadhaar photo (base64 JPEG)
    "address": {
      "state": "Maharashtra",
      "district": "Pune",
      "pincode": "411045"
    },
    "referenceId": "XXXX XXXX 4523"  // Masked Aadhaar (last 4 visible)
  }
}

// Fields we STORE in Firebase:
// ✅ name, dob, gender → for verification record
// ✅ referenceId → for audit (masked, not full Aadhaar)
// ❌ photo → NOT stored (privacy)
// ❌ address → NOT stored (not needed)
// ❌ Full Aadhaar number → NEVER stored

API Endpoint 3: Offline XML Processing (Server-Side)

POST /api/v1/xml/validate (Our Firebase Cloud Function)

Processing Steps

// Firebase Cloud Function: processAadhaarXML

Step 1: Receive ZIP file upload
  → Validate: file size ≤ 5MB, extension = .zip

Step 2: Extract XML from ZIP using share_code as password
  → If decryption fails → Error: XML-001

Step 3: Validate UIDAI Digital Signature
  → Parse XML → Check <Signature> element
  → Validate against UIDAI public certificate
  → If invalid → Error: XML-002

Step 4: Check XML freshness
  → Read "generatedOn" attribute
  → If older than 3 days → Error: XML-003

Step 5: Extract Demographics
  → poi (Proof of Identity) element:
    - name = poi.@name
    - dob  = poi.@dob          // DD-MM-YYYY format
    - gender = poi.@gender     // M/F/T
  → poa (Proof of Address) element:
    - state, district (optional, not stored)

Step 6: Return extracted data to app
  → Same format as OTP eKYC response

Request (multipart/form-data)

ParameterTypeRequiredDescription
xmlFileFileYesAadhaar XML ZIP file (≤ 5MB)
shareCodeStringYes4-digit numeric code (set during XML download)
artistIdStringYesFirebase UID of the registering artist

⚠️ Security Note: All API calls must use HTTPS/TLS 1.2+. Aadhaar number must be encrypted in transit using UIDAI's 2048-bit RSA public key. The raw Aadhaar number is NEVER logged or stored in our systems.
