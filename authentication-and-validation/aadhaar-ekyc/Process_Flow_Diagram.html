Process Flow Diagram

Aadhaar eKYC Verification — Complete Artist Registration Flow

BLA-27P0 CriticalSprint 1-2

1. Complete Registration + eKYC Flow

🟢 START
Artist Opens App → Taps "Register as Artist"
↓
📝 Step 1: Basic Information
Full legal name, email, phone number, city, Solo/Band toggle, category selection
(Singer / DJ / Dancer / Rapper / Beatboxer / Comedian / Instrumentalist / Band)
↓
🔐 Step 2: Identity Verification
User chooses verification method
↓
PATH A: Aadhaar OTP
↓
Enter 12-Digit Aadhaar
Real-time format validation
Must be exactly 12 digits, no letters
↓
🔗 API: UIDAI OTP Generate
POST /api/v1/otp/generate
Params: uid, txnId, appId, timestamp, consent
↓
Enter 6-Digit OTP
60-second countdown timer
Max 3 attempts
Resend after timer expires
↓
🔗 API: UIDAI OTP Verify
POST /api/v1/otp/verify
Returns: name, DOB, gender, photo
OR
PATH B: Offline XML
↓
Show Instructions
Guide: Download XML from
resident.uidai.gov.in
↓
Upload ZIP File
Max 5MB, .zip format only
File picker opens
↓
Enter 4-Digit Share Code
Provided during XML download
Used to decrypt the ZIP
↓
🔒 Server: Decrypt + Validate
Firebase Cloud Function
Validate UIDAI digital signature
Check XML not expired (>3 days)
Extract: name, DOB, gender
↓ Both paths merge ↓
📊 Extract Demographics
Name, Date of Birth, Gender from UIDAI response/XML
Calculate name match score (form name vs Aadhaar name)
↓
⚖️ AGE CHECK
Calculate age from DOB: (current date - DOB) in years
Is age ≥ 18?
↓
✅ AGE ≥ 18
↓
Full Artist Registration
Proceed to Step 3: Legal Consent
Copyright declaration, IPRS, data consent
↓
Step 4: Profile Setup
Stage name, bio, genre tags,
social links, portfolio video
↓
💾 Save to Firestore
artists/{uid}
kycVerification.isVerified = true
profileStatus.status = "pending_review"
↓
⏳ Admin Review Queue
Admin reviews: KYC data, name match,
content check, social link validation
↓
Approved
🎉 Profile LIVE
✅ Verified badge
📍 Visible on map & search
📱 Push notification sent
Rejected
❌ Rejected
Notification with reason
Option to re-apply
Re-verify
🔄 Re-verify
Asked to redo KYC
Data mismatch detected
❌ AGE < 18
↓
🔞 Restricted Fan Mode
User CAN enter the app
but with age-appropriate content only
↓
What they CAN do:
✅ Browse age-appropriate events
✅ Follow artists
✅ View general artist profiles
✅ Attend family-friendly events
↓
What they CANNOT do:
❌ Register as performing artist
❌ See nightlife/bar/pub events
❌ See 18+ tagged events
❌ Create events
↓
💾 Save to Firestore
users/{uid}
ageTier = "under_18"
dateOfBirth = DOB
contentFilter = "age_appropriate"
↓
🎂 When user turns 18 (based on stored DOB), system auto-upgrades access tier to full on next app open.

2. Error Handling Flow

Invalid Aadhaar (K-200)
"Please check your Aadhaar number"
→ Re-enter
OTP Expired (K-544)
"OTP expired. Request new one"
→ Resend OTP
Wrong OTP (K-569)
"Incorrect OTP. X attempts left"
→ Retry (max 3)
Max Retries Exceeded
"Too many attempts. Wait 30 mins"
→ Cooldown timer
Mobile Not Linked (K-100)
"Use Offline XML instead"
→ Switch to XML path
Invalid XML (XML-002)
"File not verified. Download fresh"
→ Re-download from UIDAI
